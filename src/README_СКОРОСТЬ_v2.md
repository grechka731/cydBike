## Честная физика (важно понимать)
Получить КРЕЙСЕРСКУЮ скорость из одного акселерометра в принципе невозможно:
при равномерном движении ускорение = 0, интегрировать нечего, любой остаточный
шум уплывает в дрейф. Поэтому здесь не "чистка интегратора", а ГИБРИД из 5
независимых источников, которые закрывают слабые места друг друга:

| Метод | За что отвечает | Слабое место (закрыто другими) |
|------|------------------|-------------------------------|
| A. Комплементарный фильтр вектора g (гиро вращает g, аксель медленно правит) | чистое продольное ускорение, не искажает наклон | сам по себе скорость не даёт |
| B. Интегратор продольного ускорения | РАЗГОН и ТОРМОЖЕНИЕ (быстрая динамика) | дрейфует на дистанции |
| C. **Вибрационный оценщик** (полоса 6..30 Гц + RMS) | КРЕЙСЕР, БЕЗ дрейфа | нужна калибровка; молчит на идеально гладком треке |
| D. ZUPT по дисперсии/гиро/вибрации | обнуление на остановках, переучёт нулей | — |
| E. Барометр BMP180 | склон => точно едем (давит ложный ZUPT) | медленный |

Фузия B+C — комплементарная: интегратор = быстрое предсказание, вибро-оценка =
медленная коррекция, гасящая дрейф.

## Ключевой компромисс
В ДВИЖЕНИИ коррекция вектора g сделана МЕДЛЕННОЙ (`MPU_GRAV_CORR_RATE = 0.30`),
иначе фильтр "съедает" длительный разгон (по |a|≈g разгон неотличим от наклона).
В ПОКОЕ коррекция быстрая (`MPU_GRAV_CORR_RATE_STILL = 5.0`).

## ОБЯЗАТЕЛЬНАЯ калибровка вибро-оценщика (1 раз)
Без неё скорость на крейсере будет в условных единицах.
1. Прошей, прокатись с РОВНОЙ известной скоростью (например 20 км/ч по GPS телефона).
2. В Serial-логе смотри поле `rms` (печатается каждые 200 мс).
3. Усреднённое `rms` на этой скорости впиши в `VIB_RMS_AT_REF` в config.h,
   а саму скорость — в `VIB_REF_KMH`. Перепрошей.
(Аналогично можно уточнить `VIB_HP_HZ/VIB_LP_HZ` под свою раму/колёса/покрышки.)

## Результат симуляции (ride: стоп→разгон→20с крейсер→тормоз→стоп)
После калибровки итоговая (фузия) скорость устойчиво держит крейсер и НЕ уплывает
в ноль, тогда как голый интегратор на крейсере показывает ~0 (доказательство, что
без метода C акселерометр-онли безнадёжен). Остановки/торможение дают ровный 0.

## Если хочешь максимум точности
Добавь реальный датчик скорости колеса (геркон/холл на спице) — это единственный
способ получить абсолютно точную скорость. Текущий код оставляет пин RX (GPIO3)
свободным под него; гибрид можно использовать как дополнительный/резервный канал.

## Новые геттеры (для вывода на экран при желании)
- `getVibSpeedKmh()`        — скорость по вибрации
- `getIntegratorSpeedKmh()` — скорость только по интегратору
- `getSpeedConfidence()`    — уверенность вибро-оценки 0..1
- `isSpeedMoving()`         — едем/стоим
